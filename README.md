# TLMC 一键分轨工具（简体中文）

English version: [README_EN.md](README_EN.md)

> 说明：本 README 内容主要是根据 `task.md` vibed 出来的用户文档，目标是给普通用户快速上手。

## 这是什么

这是一个给东方同人专辑用的“一键分轨”工具。  
它会在当前目录下扫描社团文件夹，解压 `.rar` 专辑包，读取 `.cue`，把整轨 `.flac` 切成单曲，并写入常见标签。

## 使用前准备

1. 把可执行文件放在一个“总目录”中。
2. 总目录下每个子目录代表一个社团（`[CIRCLE_NAME]`）。
3. 每个社团目录里放 `.rar` 专辑文件。
4. 建议先备份一份原始资源（尤其是首次使用时）。

目录示意：

```text
总目录/
  社团A/
    2026.04.28 [ABCD-0123] foo bar [c123].rar
  社团B/
    2025.10.01 [WXYZ-0001] album name.rar
  tlmc.exe
```

## 如何使用

1. 双击运行程序（Windows）。
2. 等待处理完成。
3. 打开 `verbose.log` 和其他审计文件，确认是否有需要人工处理的专辑。

程序特性：

- 不需要命令行参数
- 处理信息同时输出到屏幕和 `verbose.log`
- 出错会记到 `error.log`，尽量不中断整体流程

## 转换后会发生什么

- `.rar` 会被解压到同名目录（去掉 `.rar` 后缀）。
- 会查找并配对 `.flac` 与 `.cue`。
- 成功分轨后生成：`TRACK_ID - TRACK_NAME.flac`
- 处理过的原始 `.flac` 和 `.cue` 会改名为 `*.old`
- 多盘（多组 flac-cue）时会放入子目录（按原 flac 文件名分开）

## 审计文件说明（重点）

下面这些文件都会生成在程序执行目录，且**每条路径都是相对路径**，便于人工排查。

- `verbose.log`
  - 详细处理日志（也会输出到控制台）
  - 包含专辑处理过程、配对结果、审计记录等
- `error.log`
  - 错误日志（例如某个专辑/某个配对失败）
  - 先看这个文件定位硬错误
- `corrupt-cuesheet.txt`
  - cue 解析失败的条目
  - 这些通常无法自动分轨，需要你检查/替换 cue
- `multi-disc.txt`
  - 一个专辑目录内存在多组 flac-cue 配对
  - 一般需要你手动确认盘号/专辑名是否符合你的整理规范
- `missing-cue.txt`
  - 找到 flac 但没有可配对 cue
  - 需要补 cue 或人工分轨
- `missing-flac.txt`
  - 找到 cue 但没有可配对 flac
  - 需要补 flac 或清理无效 cue
- `missing-info.txt`
  - 分轨时发现标题/演出者等关键信息缺失
  - 建议后续用标签工具人工补齐

## 全部跑完后你该做什么

建议按这个顺序检查：

1. 先看 `error.log`，确认是否有失败项目。
2. 看 `corrupt-cuesheet.txt`，修复或替换损坏 cue 后可重跑。
3. 看 `missing-cue.txt` / `missing-flac.txt`，补齐缺失文件。
4. 看 `multi-disc.txt`，确认多盘目录与命名是否符合预期。
5. 看 `missing-info.txt`，补全曲目标签（标题、表演者等）。
6. 抽查若干专辑试听，确认切点与曲目顺序正确。
7. 确认没问题后，再决定是否删除 `*.old` 原始文件。

## 注意事项

- 程序会重命名原始 `flac/cue` 为 `*.old`，请先确认你有备份策略。
- 建议在整理副本上运行，避免直接对唯一原始库操作。
- 如果想“修完问题再跑一遍”，可以保留日志文件作为待办清单。
